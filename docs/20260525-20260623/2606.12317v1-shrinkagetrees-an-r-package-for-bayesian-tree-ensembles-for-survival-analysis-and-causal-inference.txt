# ShrinkageTrees: An R Package for Bayesian Tree Ensembles for Survival Analysis and Causal Inference 

by Tijn Jacobs Department of Mathematics, Vrije Universiteit Amsterdam, Amsterdam, The Netherlands 

t. jacobs@ vu. nl 

Abstract ShrinkageTrees is an R package for Bayesian tree ensembles in survival analysis and causal inference. The package implements Bayesian additive regression tree models for right- and interval-censored survival outcomes within an accelerated failure time (AFT) framework, with optional decomposition into prognostic and treatment-effect components for causal inference. Two complementary forms of regularisation are available: regularisation of the tree structure, via depth-penalising priors and Dirichlet splitting priors, and regularisation of the step heights, via global–local shrinkage priors. ShrinkageTrees provides the first implementation of the Horseshoe Forest, which places a horseshoe prior on the step heights. These regularisation strategies extend Bayesian tree ensembles to high-dimensional settings. An efficient Rcpp backend, multi-chain MCMC, and S3 methods support the full workflow: fitting, prediction, causal effect estimation, and convergence diagnostics. 

# 1 Introduction 

Survival analysis in high-dimensional settings poses three interrelated challenges. First, the relation-ship between covariates and the time-to-event outcome is often complex and unknown, calling for flexible non-parametric models. Second, the outcome may be observed imprecisely: right-censored when a patient is lost to follow-up, or interval-censored when the event is known only to have occurred between two assessment times. Third, when a treatment is involved, even the average causal effect is challenging to estimate under censoring and confounding. The effect may also vary across patient subgroups, and disentangling this heterogeneity from baseline patient characteristics requires dedicated methodology. These challenges arise routinely in clinical trials, observational studies, and genomics, often simultaneously. Bayesian Additive Regression Trees [BART; Chipman et al. (2010)] offer a natural starting point. BART models the response as a sum of many shallow decision trees, where each tree contributes a small piece of the overall prediction. The ensemble captures non-linearities and interactions without requiring a parametric form. Existing BART implementations leave gaps in each of the three areas above. Currently, no R package provides Bayesian tree ensembles for interval-censored survival outcomes. The τ -learner is a dedicated causal forest architecture that separates the prognostic function from the treatment effect (Hahn et al., 2020; Caron et al., 2022). It has not been implemented for time-to-event endpoints. Standard BART’s regularisation, while adequate in moderate dimensions, does not adapt well to the high-dimensional settings ( p ≫ n)common in genomics. 

ShrinkageTrees addresses all three gaps. The package fits Bayesian tree ensembles to right- and interval-censored survival data via accelerated failure time (AFT) models, with optional τ -learner decomposition for causal inference. The package implements two complementary regularisation strategies for high-dimensional settings. Regularisation of the tree structure encourages the ensemble to split on a subset of relevant variables. The package implements the Dirichlet splitting approach of DART (Linero, 2018). Regularisation of the step heights targets the contributions of individual leaves: it shrinks uninformative leaves towards zero while preserving strong signals. ShrinkageTrees provides the first implementation of the Horseshoe Forest (Jacobs et al., 2025), which places a horseshoe prior (Carvalho et al., 2010) on the step heights to adaptively shrink uninformative leaves. The package is implemented with an efficient C++ backend via Rcpp (Eddelbuettel and François, 2011) and provides a coherent S3 interface with print() , summary() , predict() , and plot() 

methods. The paper is aimed at both practitioners encountering Bayesian tree ensembles for the first time and experienced users seeking functionality not available in existing packages. To give a first impression, we use the ovarian dataset shipped with the package: a semi-synthetic cohort with covariates drawn from The Cancer Genome Atlas ovarian cancer cohort [TCGA-OV; The Cancer Genome Atlas Research Network (2011)]. It comprises 357 patients with right-censored

The τ -learner for causal inference 

BART has become a popular tool for causal inference (Hill, 2011; Dorie et al., 2019). The BART formulation in equation (1) estimates a single function of the covariates, which we call the prediction model . Prediction models support two simple causal-inference workflows: an S-learner includes the treatment indicator as a covariate, and a T-learner fits separate models per treatment arm (Caron et al., 2022). ShrinkageTrees takes a different approach: the τ -learner. This is the decomposition proposed by Hahn et al. (2020) as Bayesian Causal Forests (BCF) and later termed the τ -learner by Caron et al. (2022). The bcf and stochtree packages implement the τ -learner for continuous outcomes. ShrinkageTrees extends it to survival endpoints. We write Ai ∈ { 0, 1} for the binary treatment indicator and ˆe(xi) = ˆP (Ai = 1 | xi) for the estimated propensity score. The model is: 

yi = μ(xi, ˆe(xi)) + Ai τ (xi) + εi, (8) where μ is a prognostic function modelled by a forest of mμ trees and τ is a treatment effect function modelled by a separate forest of mτ trees. This separation allows the prognostic and treatment effect components to have different regularisation and complexity. The conditional average treatment effect is CATE (x) = τ (x), and the average treatment effect is ATE = E[τ (X)] .The way treatment is coded in the τ -learner is a modelling convention with several common choices. Let bi denote the coding applied to subject i. The model becomes: 

yi = μ(xi, ˆe(xi)) + bi τ (xi) + εi. (9) Hahn et al. (2020) discuss several choices, all of which are implemented in ShrinkageTrees through the treatment_coding argument. Centred coding sets bi = Ai − 1/2 and is the default. Binary 

coding sets bi = Ai. Adaptive coding sets bi = Ai − ˆe(xi), the parameterisation used in the bcf 

package. The invariant (parameter-expanded) coding of Hahn et al. (2020, Section 5.2) is invariant to the labelling of treatment and control. It reparameterises the model with group-specific intercepts and recovers τ as a difference. We refer the reader to Hahn et al. (2020) for the full construction. Figure 4 summarises the decomposition. 

Posterior computation 

All models are fitted using Markov chain Monte Carlo (MCMC). The outer loop is a Gibbs sampler that cycles through updates of the tree topologies, the step heights, and the error variance σ2. When applicable, it also updates the shrinkage parameters and censored event times. Trees are updated one at a time via Bayesian backfitting (Hastie and Tibshirani, 2000): at each iteration, a partial residual is formed by subtracting the contributions of all other trees from the response, and tree j

is then updated conditional on the current state of all other trees. The tree topology is modified through Metropolis–Hastings proposals that either grow a terminal node into an internal node with two new leaves, or prune an internal node back to a leaf. Details of the sampler are given in Kapelner and Bleich (2016). Under the standard BART and Dirichlet (DART) priors, where the step heights have a conjugate normal prior, the marginal likelihood of a proposed tree can be computed in closed form and the step heights are integrated out analytically. When global–local shrinkage priors such as the horseshoe are placed on the step heights, this conjugacy is lost: the local and global scale parameters λℓ and 

6τ must be sampled explicitly, and the tree proposals become reversible jump MCMC moves. The details of this reversible jump construction are given in Jacobs et al. (2025).

# 5 Package architecture 

ShrinkageTrees is available on CRAN and on GitHub at https://github.com/tijn-jacobs/ShrinkageTrees .The R layer wraps a C++ backend through Rcpp (Eddelbuettel and François, 2011). We organise the description as follows: (i) fitting functions and S3 classes, (ii) the Rcpp backend, and (iii) workflow features such as multi-chain MCMC, coda integration, and plotting. A testthat (Wickham, 2011) suite covers the fitting functions and user-facing methods. A vignette on CRAN demonstrates a full workflow. 

Fitting functions and S3 classes 

The package provides two main fitting functions, one for prediction and one for causal inference. It also provides six convenience wrappers that fix particular model configurations. The wrappers are named after the methods they fit (e.g., HorseTrees() , SurvivalBART() ), so users who come with a method in mind can call it directly without configuring prior_type or outcome_type by hand. Table 2 lists all eight functions. The main functions accept arguments that control the outcome type (outcome_type , timescale ), prior specification ( prior_type , local_hp , global_hp , and Dirichlet hyperparameters a_dirichlet , b_dirichlet , rho_dirichlet ), tree structure ( number_of_trees ,

power , base , p_grow , p_prune ), and MCMC settings ( N_post , N_burn , n_chains ), among others. Each wrapper preselects a subset of these to expose a smaller, task-specific interface. 

ShrinkageTrees() is the main prediction model function. It accepts a prior_type argument that selects the step height prior: "horseshoe" (per-tree global scale), "horseshoe_fw" (forest-wide global scale), "half-cauchy" (local shrinkage only), "standard" (conjugate normal, as in classical BART), or "dirichlet" (Dirichlet splitting, as in DART). CausalShrinkageForest() is the main causal-inference function. It mirrors this design for the τ -learner with independent prior config-urations for the prognostic and treatment effect forests. The six wrappers preselect a particular 

prior_type and/or outcome_type to give a smaller, task-specific interface: HorseTrees() and 

CausalHorseForest() fix the prior to the horseshoe and expose a single tuning parameter k con-trolling the half-Cauchy scales via ατ = αλ = k/ √m. SurvivalBART() and SurvivalDART() fix the AFT survival outcome with the conjugate-normal and Dirichlet priors, respectively. SurvivalBCF() 

and SurvivalShrinkageBCF() do the same for the τ -learner causal model. All prediction model functions return an object of class "ShrinkageTrees" and all causal model functions return "CausalShrinkageForest" . Both classes provide print() , summary() , predict() ,and plot() methods. The summary() method reports posterior summaries for σ and variable inclu-sion probabilities. For causal models, it also reports the ATE with a credible interval. The predict() 

method re-runs the MCMC sampler on the new covariate matrix using the training data and hyper-parameters stored in the fitted object. It always generates posterior samples internally, regardless of whether store_posterior_sample was set during fitting. The returned ShrinkageTreesPrediction

Rcpp backend 

We implement the computationally intensive components of the Gibbs sampler in C++ via Rcpp .The C++ layer covers the Metropolis–Hastings updates for the tree topology (grow, prune, and change moves), the conditional updates for the step heights, the augmentation steps for the local and global scale parameters under horseshoe shrinkage, the update for σ, and the data augmentation for censored event times. The R layer handles input validation, data preprocessing (centring, scaling, log-transformation of survival times), hyperparameter calibration, S3 class construction, and post-processing (back-transformation of predictions). The C++ backend dispatches between two forest implementations depending on the step height prior. The Forest class implements reversible jump moves that update the tree and its step heights jointly. It is used for the "horseshoe" , "horseshoe_fw" , and "half-cauchy" priors. The 

StanForest class implements moves that update the tree structure and the step heights sequentially. It is used for the "standard" and "dirichlet" priors. The latter approach is simpler and faster, but applies only when the step height prior is conjugate and global scale updates are unnecessary. A single C++ class, ScaleMixture , holds the global–local step height prior and is queried by the tree updates whenever a new step height is proposed. The current implementations are 

FixedVariance (conjugate normal, used by BART and DART), HalfCauchy (local shrinkage only), 

Horseshoe (per-tree global–local shrinkage), and Horseshoe_fw (forest-wide global shrinkage). The user’s prior_type argument selects the implementation at run time. Because the tree update and reversible jump routines interact with the prior only through the ScaleMixture interface, a developer can add a new global–local shrinkage prior by adding a new implementation and registering it. Examples include a regularised horseshoe or a Dirichlet–Laplace prior. The sampler logic does not change. Figure 8 summarises the package architecture. 

Multi-chain MCMC 

All fitting functions accept an n_chains argument. When n_chains > 1 , the package dispatches independent chains in parallel via parallel::mclapply() . On Windows, mclapply() falls back to sequential execution. The package pools the draws from all chains and recomputes posterior summaries; the returned object is a standard "ShrinkageTrees" or "CausalShrinkageForest" 

instance, so all downstream methods work without modification. Per-chain acceptance ratios and σ

traces remain available for convergence assessment.

Convergence diagnostics via coda 

We assess MCMC convergence via the coda package (Plummer et al., 2006). The package exposes posterior draws to coda through an as.mcmc.list() S3 method. The method extracts the pos-terior draws of σ, splits them by chain, wraps each chain in a coda::mcmc object, and returns a coda::mcmc.list . coda is a suggested rather than required dependency; the package’s other functionality does not depend on it. When coda is installed, summary() reports the effective sample size (ESS) and, for multi-chain fits, the Gelman–Rubin ˆR statistic. Users who require more detailed diagnostics can pass the mcmc.list object directly to any coda function, including geweke.diag() ,

heidel.diag() , autocorr.plot() , and gelman.plot() .

# 6 Related packages 

Several R packages implement Bayesian tree ensembles. We review five comparator packages and contrast their capabilities with ShrinkageTrees . Table 3 provides a summary. 1

BART (Sparapani et al., 2021) is the most general-purpose package in this comparison and covers the widest range of outcome types. It supports continuous, binary, and multinomial outcomes, and is the only other CRAN package to offer Bayesian tree ensembles for survival data. Survival models include a discrete-time hazard formulation ( surv.bart() ), an AFT formulation ( abart() ), competing risks ( crisk.bart() ), and recurrent events ( recur.bart() ). The package also supports Dirichlet splitting priors via sparse = TRUE . BART does not support global–local shrinkage priors on the step heights, causal forest models, or interval-censored survival outcomes. Multi-chain MCMC requires the prefixed mc.* wrappers (e.g. mc.abart() ). 

dbarts (Dorie et al., 2026) provides a BART sampler for continuous and binary outcomes. Its key feature is an updatable sampler object: predictors and responses can be modified mid-sampling, which makes dbarts well suited as a conditional model within a larger Gibbs sampler. The package supports multi-chain MCMC but does not support survival models, τ -learner models, or global–local shrinkage on the step heights. 

bcf (Hahn et al., 2020) implements the τ -learner with separate prognostic and treatment-effect forests for continuous outcomes. It places half-Cauchy and half-Normal hyperpriors on the leaf scales of the two forests, respectively. The package does not support survival endpoints, global–local shrinkage priors, or Dirichlet splitting. 

stochtree (Herren et al., 2026) implements BART, XBART (He et al., 2019), the τ -learner, and XBCF for continuous, binary, and ordinal outcomes. It supports variance forests for heteroskedastic models and serialises fitted objects so they can be reused as warm starts. stochtree does not support survival models, Dirichlet splitting priors, or global–local shrinkage on the step heights. 

SoftBart (Linero and Yang, 2018) replaces the hard axis-aligned splits of standard BART with smooth logistic splitting rules, so the posterior mean is a continuous function of the covariates. Regularisation is achieved through a Dirichlet prior on the variable inclusion probabilities (the same mechanism as DART) rather than through priors on the step heights. This combination can improve performance when the true regression surface is smooth, but introduces additional computational cost. ShrinkageTrees retains hard decision splits, as in classical BART, and instead places shrinkage priors on the step heights themselves. SoftBart does not support survival endpoints, the τ -learner, or global–local shrinkage on the step heights. Across these packages, ShrinkageTrees occupies a distinct niche: survival outcomes (right- and interval-censored), the τ -learner for survival, and global–local shrinkage on the step heights.

# 7 Discussion and conclusion 

ShrinkageTrees brings global–local shrinkage priors to Bayesian tree ensembles for survival analysis and causal inference. The package separates regularisation of the tree structure from regularisation of the step heights, so practitioners can tailor the regularisation strategy to the problem at hand. It is the first CRAN package to offer the τ -learner for survival endpoints, horseshoe priors on tree ensemble step heights, and interval-censored survival outcomes within a single framework. The package has several limitations that suggest directions for future work. All covariates must be numeric. The axis-aligned splitting rules handle continuous, binary, and ordered categorical variables (encoded as integers) directly, but the user must dummy-encode unordered categorical variables. This can be suboptimal for high-cardinality factors, where native subset splits would be more efficient. The current AFT formulation assumes normally distributed errors. Alternative parametric error distributions, such as the logistic or log-logistic, would broaden the package’s applicability. Non-parametric error models, for example those built on stick-breaking process priors, are a further direction. 

ShrinkageTrees is available on CRAN and on GitHub at https://github.com/tijn-jacobs/ ShrinkageTrees , where bug reports and feature requests can be filed via the issue tracker at https: //github.com/tijn-jacobs/ShrinkageTrees/issues . Jacobs et al. (2025) develop the underlying methodology.
