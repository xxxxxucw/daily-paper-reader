## Abstract 

Task decomposition aims to transform ambiguous instructions into executable atomic subtasks, thereby guiding high-precision tool retrieval. However, our analysis reveals that directly adopting tool retrieval metrics, i.e., Recall or NDCG, as rewards for task decomposition can easily induce reward hacking in reinforcement learning-based meth-ods. Specifically, models tend to maximize retrieval matching through strategies such as repetitive decomposition. This spurious correlation between the shallow features of decomposition results and retrieval metric impairs generalization in Out-of-Domain (OOD) scenarios involving unseen tools. To address this issue, we propose PCTD, a 

Preference-guided Counterfactual Task Decomposition framework. PCTD quantifies the marginal causal gain of decomposition on retrieval ranking through a counterfactual re-ward, thereby cutting off spurious correlations at their source. Meanwhile, it introduces a preference reward to impose fine-grained structural supervision on logical coherence and atomicity, encouraging the model to generate high-quality decompositions. In addition, we construct MTDTool, the task decomposition benchmark specifically designed for mobile multi-turn interactions. Extensive experiments demonstrate that PCTD allevi-ates repetitive decomposition and surpasses SOTA methods in retrieval, decomposition quality, and OOD generalization. In-Domain Out-of-Domain

## 1 Introduction 

Large language model (Zhao et al., 2026b; Naveed et al., 2025; Wang et al., 2026) driven agents are evolving into intelligent entities capable of autonomously decomposing tasks and invoking tools to achieve complex goals (Huang et al., 2024c; Xu et al., 2024). In multi-turn mobile interaction scenarios, user requests typically rely on dialogue history, coreference relationships, and implicit slots, where a single input is often insufficient to fully describe the objective. This necessitates that agents transform ambiguous instructions into semantically clear, well-bounded, and executable atomic sub-tasks to guide high-precision tool retrieval. Early prompt engineering based methods, such as ReAct (Yao et al., 2022),

However, due to the lack of end-to-end optimization mechanisms and heavy reliance on intrinsic knowledge, these methods are prone to decomposition deviations caused by cumulative errors when facing unseen tools or long-range dependencies. To overcome this limitation, reinforcement learning (RL) based end-to-end methods like ToolQP (Fang & Glass, 2026) incorporated downstream tool retrieval metrics (e.g., Recall@K, NDCG@K) into mixed reward signals. Although these approaches enhance final retrieval performance, they fundamentally rely on static outcome-oriented supervision and suffer from two core limitations. First, directly employing retrieval metrics as rewards is highly susceptible to reward hacking . Models tend to maximize matching scores through superficial strategies such as repetitive decomposition, leading to a spurious correlation between the shallow lexical features of decomposition results and retrieval outcomes. This severely impairs generalization in Out-of-Domain (OOD) scenarios involving unseen tools. Second, there is a lack of preference modeling for decomposition structure. plan-level similarity-based rewards (e.g., the reward signals utilized in ToolQP) fail to constrain structural quality dimensions such as logical coherence, context faithfulness, and atomicity, further exacerbating the issue of repetitive decomposition. Empirical results in Figure 1 validate this phenomenon: existing methods perform reasonably well on in-distribution data but suffer significant performance degradation on OOD datasets involving unseen tools (e.g., ToolQP drops from 83.6 to 74.3). Meanwhile, a clear negative correlation is observed between repetition rate and performance (as the repetition rate decreases from 5.2% to 0.7%, performance improves to 91.2). Therefore, mitigating the spurious correlations and repetitive decomposition induced by reward hacking is critical for improving model performance. To address these dual challenges, we propose PCTD, a joint optimization framework for task decom-position and tool retrieval tailored for mobile multi-turn interactions. This framework introduces a preference reward Rpref , which leverages a trained reward model to score candidate decompositions against human references conditioned on context, providing fine-grained structural quality supervision via relative score differences; simultaneously, it incorporates a counterfactual reward Rcf that uses the retrieval results of the raw query as a baseline to quantify the marginal gain of the decomposition strategy on tool ranking, thereby offering causal feedback for decomposition behavior. Furthermore, existing benchmarks such as HammerBench (Wang et al., 2025) and ToolBench (Qin et al., 2024) lack fine-grained process-level annotations, and there is currently no dataset specifically designed for evaluating the task decomposition process, we constructed MTDTool. This benchmark is designed for complex multi-turn interaction scenarios in mobile environments and provides fine-grained evaluation support. Unlike existing datasets that primarily focus on single-turn instructions or final execution outcomes, MTDTool automatically annotates dialogue state evolution, context-aware intent rewriting, and atomic task se-quences for each conversational turn. The dataset highlights the dynamic evolution of intents across multi-turn dialogues and cross-application collaboration, providing a standardized basis for quantifying the impact of decomposition quality on retrieval effectiveness. The contributions of this paper are summarized as follows: ❶ Insight. We reveal the dual limitations of existing RL-based agents in task decomposition: (i) directly using retrieval metrics as rewards is prone to reward hacking, severely impairing OOD generalization with unseen tools; and (ii) the lack of structural preference modeling further exacerbates repetitive decomposition. ❷ Method. We propose PCTD, a joint optimization framework integrating counterfactual reasoning with preference modeling. It leverages counterfactual rewards to quantify the marginal causal gain of decomposition on retrieval ranking to cut off spurious correlations, while introducing preference rewards to enforce fine-grained structural supervision on logical coherence and atomicity. ❸ Dataset. We release MTDTool, the fine-grained benchmark for task decomposition in mobile multi-turn interactions. Generated via a state-machine-driven pipeline, it provides multi-level annotations for dialogue state evolution, intent rewriting, and atomic task sequences, filling the gap in process-oriented evaluation. ❹ Evaluation. Extensive experiments on MTDTool and mainstream benchmarks demonstrate that PCTD significantly outperforms SOTA methods in tool retrieval, decomposition quality, and OOD generalization.

2.1 Problem Formulation: Multi-Turn Task Decomposition and Tool Retrieval 

We formulate multi-turn agent interaction as a sequential decision process. At turn t, the agent receives a user query qt and the dialogue history Ht = {q1, o1, . . . , qt−1, ot−1}, forming the contextual input 

xt = ( Ht, qt). Given a large-scale tool library T = {a1, . . . , aM} containing M APIs, the agent employs a policy model πθ to generate a natural-language task decomposition sequence yt ∼ πθ (· | xt). The generated sequence is parsed into a set of semantically clear atomic subtasks At = {τ1, . . . , τL}, which aims to mitigate semantic ambiguity, implicit slots, and cross-turn coreference in direct retrieval. Given a 2top-K tool retriever RK (·), each atomic subtask is used to retrieve candidate tools, and the retrieved lists are merged, deduplicated, and truncated into a decomposition-aware ranked list: 

LKt = TopK (Dedup (Merge (RK (τ1), . . . , RK (τL)))) . (1) Let Gt ⊆ T denote the ground-truth tool set required for the contextual input xt. Our objective is to optimize the policy πθ such that the generated decomposition yt maximizes the ranking consistency between the retrieved list LKt and the ground-truth tool set Gt.

3.1 Dataset Construction via a State-Machine-Driven Framework 

To address the evaluation needs for fine-grained task decomposition in mobile multi-turn tool invocation, this paper proposes an automated dialogue generation framework driven by a state machine. This 3framework models multi-turn interactions as a stochastic decision process driven by state evolution, uniformly characterizing context transitions, task decomposition, and tool execution behaviors. In contrast to traditional construction paradigms that focus solely on input-output pairs, our method explicitly models the dynamic dependencies among dialogue states, user intents, and tool calls, thereby generating high-fidelity multi-turn corpora with fine-grained structural annotations. 

3.1.1 State Space Definition and Dynamic Evolution Mechanism 

As illustrated in Figure 2, we model the construction of multi-turn dialogues as a state-driven stochastic evolution process. Let S denote the dialogue state space, which contains 11 canonical interaction behaviors (see Table 10), including task execution states (e.g., success s1, in-progress s2, and failure s3), interaction states (e.g., clarification s4 and selection s5), and intent-management states (e.g., silence s6,cancellation s7, revocation s8, supplementation s10 , and null s11 ). Given a tool library T , each tool aj is associated with a semantically compatible state subset Saj ⊆ S ., which constrains the feasible interaction states of that tool. During initialization, each tool’s state is sampled from a tool-conditioned prior: 

s0, i ∼ Pinit (· | ai), s0, i ∈ S ai , (3) where Pinit controls the prior frequency of basic interaction phenomena. We further impose lightweight constraints to ensure that the generated dialogues contain sufficient multi-turn interaction signals, such as clarification, continuation, or silence. Subsequently, the evolution process comprises context decision, intent transition, and state transition. At turn t, the framework first samples a context decision 

ct ∼ Pctx (· | t), ct ∈ { inherit, switch }, (4) which determines whether the dialogue continues the previous context or switches to a new topic. For each active tool aj, the framework then samples a user-side pending intent and derives the corresponding execution state: 

pt,j ∼ Pintent (· | st−1, j, ct), (5) 

st,j ∼ Pstate (· | pt,j, st−1, j), st,j ∈ S aj . (6) This decoupled formulation separates user intent evolution from assistant execution feedback, making the generation process more controllable. Finally, we apply a lifecycle update to the active tool set: if both the pending intent pt,j and the execution state st,j are assigned to the null state, tool aj is removed from the current context. 

3.1.2 Context Control and Task Composition Strategy 

To simulate the topic convergence characteristics of real-world dialogues, the framework introduces a context decision variable ct ∈ { inherit , switch } at each evolution node. The probability of switching topics follows an exponential decay function: 

Pswitch (t) = max (pmin , p0 · αt−1), (7) where p0 denotes the baseline probability of topic switching in the initial turn, α is the decay coefficient governing the rate of topic convergence, and pmin sets the lower bound for maintaining minimal topic diversity in later dialogue stages. This mechanism ensures high intent variability in early turns while progressively enhancing contextual coherence and stability as the dialogue progresses. At the task structure level, the system supports proportional sampling of three modes: single-tool, multi-tool collaboration, and multi-intent parallelism, while allowing dynamic injection of new tasks during context inheritance. The tool sampling phase combines global random weighting with scene-based knowledge base guidance, utilizing pre-defined or LLM-dynamically-generated scene templates to ensure logical self-consistency of tool combinations within specific domains. 

3.1.3 Dual-Channel Decoupled Generation and Quality Constraints 

Corresponding to the content synthesis module in the pipeline, the framework employs a dual-channel mechanism to generate user queries qt (as shown in Appendix prompt 1) and assistant responses ot (as shown in Appendix prompt 2) separately. The user-side generator integrates available tool lists, historical state trajectories, and linguistic style features, employing natural language strategies to avoid mechanical concatenation and explicitly completing missing information dimensions in clarification states. The assistant-side generator strictly adheres to state-semantic mapping rules, ensuring precise alignment between response content and current execution states (e.g., success confirmation, failure attribution). To guarantee corpus quality, we embed a structured hard-constraint layer into the generation process, including: (i) the one-tool-one-request atomicity principle; (ii) colloquial conversion of terminology; (iii) normalization of temporal expressions; and (iv) zero-response specifications for silence states. These constraints act as quality filters throughout the generation process, ensuring that the output data possesses both linguistic naturalness and logical rigor. 43.1.4 Fine-Grained Annotation and Evaluation Support

3.2 Joint Reward Mechanism 

In the scenario of contextual task atomization, the efficacy of model-generated decomposition results should not be determined solely by superficial textual rationality, but rather by their actual contribution to downstream tool retrieval performance and the semantic quality of the decomposition structure itself. To this end, we propose a joint reward mechanism that integrates counterfactual (Verma et al., 2024b; Guo et al., 2025b) causal feedback with preference alignment, guiding the policy model to learn optimal semantic decomposition paths. 

3.2.1 Counterfactual Baseline Construction and Reward Mechanism 

To characterize the true contribution of task decomposition to tool retrieval, we construct a Structural Causal Model (SCM), as illustrated in Figure 3. Specifically, X denotes the multi-turn dialogue input, 

D denotes the task decomposition strategy, Y denotes the retrieval ranking quality, and Z represents confounding factors such as high-frequency keywords, tool priors, and model parametric memory. Traditional methods directly optimize outcome-oriented metrics such as Recall or NDCG, which are susceptible to the spurious path X → Z → Y. As a result, retrieval gains caused by raw-query keywords or tool priors may be incorrectly attributed to task decomposition, inducing shallow strategies such as repetitive decomposition and keyword stacking. To mitigate this attribution bias, we introduce a paired counterfactual comparison. Given the same input, tool library, retriever, and ground-truth tool set, we jointly evaluate the decomposition branch d and the non-decomposition baseline d0. Since the two branches share the same retrieval environment, confounding factors such as keyword priors, tool distributions, and parametric memory are held fixed and subtracted out through differencing. Therefore, the residual performance difference can be used as a controlled estimate of the marginal contribution of task decomposition. Formally, we treat task decomposition as an intervention on the retrieval process and define the marginal causal contribution of decomposition as 

∆(D; X) = E [Y | do (D = δ), X] − E [Y | do (D = δ0), X] , (9) where δ is the atomized decomposition strategy generated by the policy model, and δ0 is the non-decomposition baseline, i.e., directly retrieving tools using the raw context X. Under the paired-counterfactual assumption that the nuisance factors Z are shared between the two branches conditioned 5on the same input X, this difference provides a controlled estimate of the additional retrieval benefit brought by task decomposition. In practice, we instantiate the potential outcome Y with the retrieval ranking metric NDCG@K. Given the multi-turn dialogue context x = ( H, q) and the ground-truth tool set Gt, we first evaluate the non-decomposition branch by directly retrieving tools with the raw context:

3.3 Preference Reward Modeling 

To compensate for the semantic blind spots of pure retrieval metrics, we introduce an independent preference reward Rpref . This reward is based on a specially trained Process Reward Model (PRM) that evaluates decomposition results across five dimensions: completeness, accuracy, coreference resolution, standardized expression, and contextual consistency (detailed descriptions and penalty rules for each dimension are provided in Appendix Table 4). Let sϕ(x, y) denote the raw quality logit assigned by the PRM to a decomposition y under context x, and let y⋆ denote the human-annotated reference decomposition. For each candidate decomposition yt,i, we construct a reference-anchored preference pair (yi, y⋆) and define the preference reward as:

During GRPO optimization, we combine the counterfactual retrieval reward and the preference reward into a unified training signal:

The preference reward Rpref encourages the policy to generate logically coherent, context-faithful, and atomic decompositions, while the counterfactual reward Rcf promotes decompositions that improve tool ranking and coverage beyond the raw-query baseline. By balancing these complementary signals, the policy is optimized via Eq. 2 to produce decompositions that are both structurally sound and effective for retrieval. 4.1 Evaluation Benchmarks 

We evaluate PCTD on two benchmarks: ToolRet (Shi et al., 2025) and MTDTool. ToolRet aggregates 35 widely-used tool-calling datasets spanning three domains (Web, Code, and Custom), with a total corpus of 44K tools. MTDTool is a dataset specifically constructed for complex multi-turn mobile interaction scenarios, covering 237 vertical-domain tools across diverse mobile contexts. The training and test splits of ToolRet strictly follow the setup in ToolQP: we sample 10k instances from Web-domain data and synthesize tool-calling trajectories using Qwen3.6-Plus to form the training set; among these, 3.1k Web samples serve as the In-Domain test set, with all involved tools exposed during training, while the remaining test data in ToolRet constitute the Out-of-Domain test set. For MTDTool, we partition the test set into In-Domain and OOD subsets based on tool vertical domains. We use NDCG@K and Completeness@K (1 [R@K = 1]) as evaluation metrics, denoted as N@K and C@K in this paper. Detailed specifications are provided in the Appendix D.1. 

4.2 Baselines 

To comprehensively evaluate task decomposition and tool retrieval performance, we compare against three representative categories of baselines: (1) closed-source LLMs (ChatGPT-5.0 1, Qwen3-series (Yang et al., 2025), MiniMax-M2.5 2, and DeepSeek-V4-Pro 3), which serve to establish the upper bound of general-purpose models in zero-shot or few-shot decomposition scenarios; (2) prompting-based methods (Q2E (Wang et al., 2023), ReInvoke (Chen et al., 2024), ToolReAGt (Braunschweiler et al., 2025), and PLUTO (Huang et al., 2024b)), which optimize decomposition strategies via explicit chain-of-thought or structured retrieval augmentation, representing lightweight paradigms without parameter updates; and (3) the RL-based end-to-end optimization method ToolQP (Fang & Glass, 2026), which incorporates downstream tool retrieval metrics into mixed reward signals to jointly optimize decomposition and retrieval processes. Detailed descriptions of all baselines are provided in the Appendix A. Detailed implementation settings are provided in the Appendix B. 

4.3 Main Results 

Table 1 presents a comprehensive performance comparison between PCTD and various baseline methods on the ToolRet and MTDTool benchmarks, yielding the following key findings. First , to address the OOD generalization failure of existing RL methods caused by their reliance on surface keyword co-occurrences, PCTD demonstrates significant robustness by cutting off spurious correlations via counterfactual rewards. Under the OOD setting of MTDTool with Qwen3-8B as the base model, the N@10 of PCTD reaches 82.74, an improvement of 4.82 over the second-best RL baseline ToolQP (77.92). It also achieves an N@10 of 39.82 on the ToolRet benchmark under the OOD setting, outperforming ToolQP (37.77). This directly proves that the model has genuinely learned the causal logic for guiding retrieval rather than merely memorizing the tool distribution of the training set, thereby effectively mitigating performance collapse in scenarios involving unseen tools. Second , addressing repetitive decomposition and reward hacking caused by the lack of structural constraints (i.e., normative constraints on task decomposition results across logical, semantic, and execution dimensions), PCTD achieves a substantial leap in decomposition quality while maintaining high retrieval accuracy. In the In-Domain scenario of MTDTool, the C@10 of PCTD (Qwen3-8B) reaches 87.08, significantly outperforming ToolQP (83.61) by 3.47. Combined with the negative correlation between the repetition rate and performance in Figure 1 (where performance increases to 91.2 as the repetition rate drops from 5.2% to 0.7%), this improvement verifies that the Rpref 

effectively eliminates reward hacking behaviors by suppressing the generation of redundant sub-tasks, enabling the model to output more precise and high-quality decomposition strategies. 4.6 Deeper Analysis When might PCTD fail? We further analyze that improper reward weight settings can lead to the failure of PCTD. As shown in Figure 4 (a)-(c), when the preference reward weight wpre f = 0, the model 9relies solely on counterfactual signals for optimization. Although it maintains high retrieval accuracy In-Domain (N@10 = 90.6), the duplication rate surges to 5.24%, and OOD performance drops sharply to 78.6. This confirms that pure causal optimization without preference constraints still induces reward hacking, resulting in generalization failure. Conversely, an excessively high wpre f may slightly sacrifice retrieval accuracy due to over-regularization, despite suppressing redundancy. Therefore, the risk of PCTD failure primarily stems from extreme imbalances in reward weights; only by maintaining a dynamic balance between causal gain and structural constraints can dual optimality in retrieval accuracy and generalization robustness be achieved. Detailed hyperparameter analysis is provided in the Appendix E.1. We further analyze the training dynamics of various metrics, comparing the existing reinforcement learning baseline relying on traditional tool retrieval rewards with our PCTD framework, as illustrated in Figure 5. Specifically, as shown in Figure 5(a), the reward model score of PCTD rises more rapidly and reaches a final value of 0.62, significantly outperforming the baseline value of 0.59. This indicates that the preference reward Rpref effectively guides the model to generate high-quality decomposition results that are more logically coherent, highly atomic, and better aligned with human references, thereby compensating for the lack of structural supervision in existing methods. In terms of policy exploration, as depicted in Figure 5(b), the policy entropy of PCTD rapidly decreases and stabilizes at a minimal level of 0.05, substantially lower than that of the baseline. This confirms that the counterfactual reward Rcf provides clear causal gradient signals, enabling the model to quickly converge to the optimal decomposition strategy and reducing ineffective exploration in the surface-level lexical space. More crucially, as demonstrated in Figure 5(c), the response length of PCTD is significantly shortened to approximately 17 tokens, whereas the baseline remains at 22 tokens. This directly verifies that PCTD successfully suppresses reward hacking behaviors, such as repetitive decomposition common in existing methods, through structural constraints, forcing the model to output refined sub-tasks. Ultimately, benefiting from the streamlined generation sequence and faster convergence, as evidenced in Figure 5(d), the per-step training time of PCTD drops to 92 milliseconds, outperforming the 110 milliseconds of the baseline. This demonstrates that while resolving generalization failures and structural redundancies, the proposed method not only avoids additional computational overhead but also achieves an optimization in overall training efficiency. 

## 5 Related Work 

5.1 Planning and Tool Retrieval in Agents 

Research on agent planning and tool retrieval has evolved along two primary paradigms (Huang et al., 2024c; Yehudai et al., 2025): (1) Prompt-based and In-Context Learning approaches, grounded in the Reasoning-Acting-Reflecting cycle established by ReAct (Yao et al., 2022) and ToolReAct (Braunschweiler et al., 2025). Subsequent works (e.g., Toolformer (Schick et al., 2023), Gorilla (Patil et al., 2024), and the ToolLLM (Qin et al., 2024) series) have achieved effective tool selection within fixed inventories by explicitly generating chains of thought, sub-goals, or API invocation sequences; and (2) Joint optimization via Retrieval-Augmented Generation (RAG) (Zhao et al., 2026a) and Reinforcement Learning (Liu et al., 2026), designed to address challenges in large-scale open tool libraries. These methods typically decompose complex instructions into structured sub-queries (e.g., DAG-based atomic planning in Plan-RAG (Verma et al., 2024a) or iterative query planning in TOOLQP (Fang & Glass, 2026)) and refine policies using synthetic trajectories and RLVR. Recent studies (e.g., ToolRL (Qian et al., 2026), RLTR (Li et al., 2025), GiGPO (Feng et al., 2026)) further incorporate GRPO (Guo et al., 2025a), fine-grained tool-use rewards, or hierarchical advantage estimation to enhance end-to-end performance in long-horizon planning and multi-step invocation. However, existing paradigms directly use retrieval metrics as rewards, which easily induces reward hacking by relying on spurious correlations between shallow lexical features of decompositions and retrieval outcomes, leading to OOD generalization failure and repetitive decomposition. To address this, we propose the PCTD framework, which cuts off spurious correlations via counterfactual rewards and synergizes preference rewards to enforce fine-grained structural constraints, achieving joint optimization of task planning and tool retrieval in both causal gain and structural quality. 

5.2 Benchmarks for Tool Retrieval 

Existing benchmarks for agent task decomposition and tool retrieval fall into two primary categories: tool-interaction benchmarks (e.g., API-Bank (Li et al., 2023a), ToolBench (Qin et al., 2024), MINT-Bench (Wang et al., 2024b), τ-Bench (Yao et al., 2024), UltraTool (Huang et al., 2024a), and ToolRet (Shi et al., 2025)), which focus on tool selection, parameter alignment, complex long-tail invocation, and retrieval robust-ness; and task-oriented benchmarks (e.g., AgentBench (Liu et al., 2024), GAIA (Mialon et al., 2024), WebArena (Zhou et al., 2024), and TaskBench (Shen et al., 2024)), which emphasize end-to-end planning, task decomposition, and completion rates in open environments. Although recent efforts like Ham-10 merBench (Wang et al., 2025) have extended evaluation to function calls in real-world mobile scenarios, mainstream benchmarks remain constrained by Outcome Bias, relying predominantly on final success rates or trajectory-level metrics. They critically lack fine-grained process annotations for multi-turn interactions, such as cross-turn context inheritance, coreference resolution, dynamic intent evolution, task atomization, and semantic tool mapping. To address this, we introduce MTDTool, a fine-grained benchmark for multi-turn mobile interactions. By providing multi-level annotations covering dialogue intent evolution chains and ranked tool candidates, it establishes a new paradigm for fine-grained process evaluation in complex mobile contexts, offering a standardized basis for quantifying the impact of task decomposition quality on tool retrieval efficacy. 

## 6 Conclusion 

In this work, we reveal that existing reinforcement learning-based task decomposition methods are prone to reward hacking due to ill-designed reward signals, leading to repetitive sub-task decomposition and severe OOD generalization failure. To address these issues, we propose PCTD, a framework that synergizes counterfactual rewards to cut off spurious correlations and preference rewards to enforce structural quality, fundamentally resolving the aforementioned challenges. Furthermore, we release MTDTool, a fine-grained benchmark for evaluating task decomposition in mobile multi-turn interactions. Extensive experiments show that PCTD effectively eliminates redundant decomposition, significantly outperforming SOTA methods in retrieval accuracy, decomposition quality, and OOD robustness.

It demonstrates strong capabilities in natural language understanding, reasoning, and text generation, enabling it to handle complex dialogue and knowledge-intensive tasks. • DeepSeek-V4 Series is a state-of-the-art open-weight large language model built on a Mixture-of-Experts (MoE) architecture with auxiliary-loss-free load balancing and FP8 mixed-precision training. It achieves superior reasoning and instruction-following performance while maintaining high computa-tional efficiency through sparse activation and optimized inference kernels. • MiniMax-M2.5 is a high-performance proprietary large language model featuring an optimized Trans-former architecture with enhanced long-context processing and multi-modal alignment capabilities. 

A.2 Prompting Methods 

• Q2E (Wang et al., 2023) is a semantic query expansion mechanism based on Large Language Models (LLMs). For short and ambiguous user queries, Q2E leverages the reasoning capabilities of LLMs to reconstruct them into content-rich pseudo-documents, thereby improving the relevance ranking performance of retrieval systems without requiring additional supervision signals. • ReInvoke (Chen et al., 2024) proposes a zero-shot tool retrieval paradigm based on Large LLMs, the essence of which is to decompose user intent into tool-executable semantic units. By reformulating raw queries into structured instructions aligned with tool descriptions, Re-invoke achieves fine-grained mapping from abstract requirements to specific tool parameters. This approach effectively bridges the semantic gap between user queries and tool interfaces without requiring fine-tuning, thereby enhancing tool retrieval performance in complex multi-task scenarios. • ToolReAGt (Braunschweiler et al., 2025) constructs an iterative retrieval framework based on the ReAct paradigm, core to which is the mandatory embedding of tool retrieval within the Thought-Action-Observation loop. For each sub-task, the Agent generates a tailored query via Thought and mandatorily invokes the retriever to obtain candidate tools. Subsequently, in the Observation phase, it evaluates matching relevance; if no suitable tool is found, it leverages feedback to enter the next iteration for query refinement. • PLUTO (Huang et al., 2024b) employs a post-retrieval planning and editing mechanism. It initially retrieves a candidate set of tools based on the user query, and subsequently refines the results through a two-step process leveraging LLMs. First, a planning module filters the most relevant tool subset from the candidates and determines their invocation order according to the current task context. Second, an editing module instantly corrects and adapts the documentation or parameter examples of the selected tools to eliminate semantic discrepancies between the original descriptions and specific task requirements. 

A.3 RL-Based Method 

• ToolQP (Fang & Glass, 2026) reformulates tool retrieval as an iterative planning process and incorpo-rates Reinforcement Learning with Verifiable Rewards (RLVR) to optimize the planning policy. During inference, the model decomposes abstract intents into structured sub-goals with specific parameters and preconditions via sub-goal decomposition, generating targeted queries accordingly. In the train-ing phase, TOOLQP establishes a closed-loop optimization mechanism: it utilizes downstream tool retrieval metrics (e.g., Recall@K or MRR) as verifiable reward signals to directly guide the parameter updates of the planner. 

A.4 Re-ranking methods 

• Qwen3-Embedding series (Zhang et al., 2025)is a high-performance text embedding model built upon the Qwen3 dense Transformer architecture, optimized specifically for semantic representation and retrieval tasks. It employs contrastive learning with hard negative mining and multi-stage training to achieve superior performance on MTEB benchmarks while maintaining efficient inference through knowledge distillation from larger teacher models. 15 • gte-Qwen2-1.5B-instruct (Li et al., 2023b) is a compact text embedding model based on the Qwen2-1.5B dense Transformer architecture, fine-tuned specifically for semantic retrieval and representation tasks. It achieves competitive MTEB performance through contrastive learning with hard negative mining and instruction-aware training, while maintaining high inference efficiency suitable for resource-constrained deployment scenarios. • e5-mistral-7b-instruct (Wang et al., 2024a) is a text embedding model built upon the Mistral-7B dense Transformer architecture, fine-tuned with instruction-aware contrastive learning for semantic retrieval tasks. It achieves strong performance on MTEB benchmarks by leveraging the base model’s reasoning capabilities and optimizing representation quality through hard negative mining and multi-stage training. ## B Implementation Details 

Our PCTD framework is trained in two stages. We apply LoRA adaptation (rank=64, alpha=128) to Qwen3-4B and Qwen3-8B base models initialized from SFT checkpoints. The training configuration uses a batch size of 128, a PPO mini-batch size of 64, and a PPO micro-batch size of 8 per GPU. The maximum prompt length is set to 1024 and the maximum response length to 128. During GRPO rollout, we sample 8 candidate responses per prompt ( n = 8) and employ vLLM as the inference engine with a GPU memory utilization of 0.6. The multi-signal reward consists of two components: a preference reward with weight wpre f and a counterfactual reward with weight 1 − wpre f .The preference reward is provided by a Qwen2.5-1.5B reward model, We set wpref = 0.5 to balance the preference reward and the counterfactual retrieval-gain reward. Within Rcf , we set λrank = λcov = 1.0 unless otherwise specified. During both training and evaluation, we use a Qwen3-Embedding-4B based tool recall service to retrieve the top-10 tools for each query. Following standard tool-recall practice, each decomposed subtask independently retrieves its top-K candidate tools, and the results are merged via round-robin ranking with deduplication.
