## 摘要
背景。大型语言模型（LLM）在理解和生成源代码方面能力日益增强，已被广泛应用于代码补全、修复和漏洞检测等软件工程任务中。然而，尽管其经验性能强大，LLM识别恶意或易受攻击代码模式的内部机制仍知之甚少。目的。我们研究了恶意代码检测行为在LLM的前馈网络（FFN）神经元中的编码位置，并通过因果干预对所识别的神经元进行归因验证。旨在识别检测恶意代码中最重要的神经元。方法。我们应用机械可解释性方法，在三个指令微调的LLM（Llama3.1-8B-Instruct、Mistral-v0.3-7B-Instruct和Qwen2.5-7B-Instruct）中定位负责恶意代码检测行为的神经元。使用来自PyPI Malregistry的1500个恶意和1500个良性PyPI包，将该行为归因于一组神经元。结果。实验结果表明，增强促进恶意代码检测的神经元同时抑制抑制性神经元可以提高准确率，而反向操作则会使预测坍缩到单一类别，尽管幅度和一致性高度依赖模型。我们证明了防护栏检测机制在不同模型间存在差异，每个模型在其FFN层中以不同方式表示其恶意代码检测行为。结论。探究与安全知识相关的神经元有助于我们深入了解LLM如何编码恶意编程概念，识别潜在的有害记忆行为，从而为开发更可靠的防御机制铺平道路，例如针对代码型LLM的神经元级编辑、选择性遗忘和安全感知对齐。

## Abstract
Background. Large language models (LLMs) have become increasingly capable of understanding and generating source code, leading to their widespread adoption in software engineering tasks such as code completion, repair, and vulnerability detection. However, despite their strong empirical performance, the internal mechanisms through which LLMs recognize malicious or vulnerable code patterns remain poorly understood. Aim. We investigated where the malware detection behavior is encoded inside LLMs Feed Forward Network (FFN) neurons and verified the attribution with causal interventions on the neurons identified. This aims to identify the most important neurons in detecting malicious code. Methods. We applied mechanistic interpretability methods to locate the neurons being responsible for malware-detection behavior in three instruction-tuned LLMs: Llama3.1-8B-Instruct, Mistral-v0.3-7B-Instruct, and Qwen2.5-7B-Instruct. Using 1,500 malicious and 1,500 benign PyPI packages from the PyPI Malregistry, we attribute the behavior to a set of neurons. Results. The experimental results reveal that amplifying facilitating neurons for malware detection while suppressing inhibiting ones can boost accuracy, while the reverse collapses predictions toward a single class, although the magnitude and consistency is heavily model-dependent. We demonstrated that the guardrail detection mechanism varies across models, each represents its malware detection behavior differently within its FFN layers. Conclusions. Probing the neurons associated with security-relevant knowledge helps us gain insights into how LLMs encode malicious programming concepts, identify potentially harmful memorized behaviors, paving the way toward more reliable defense mechanisms, such as neuron-level editing, selective unlearning, and security-aware alignment for code-focused LLMs.

## 1. 核心问题与整体含义（研究动机和背景）
大型语言模型（LLM）在代码补全、漏洞检测等软件工程任务中表现出色，但其内部如何识别恶意或易受攻击的代码模式仍不透明。当前防御策略（如提示工程、安全微调）将模型视为黑盒，难以抵御自适应或对抗性攻击。本论文旨在通过机械可解释性方法，定位 LLM 前馈网络（FFN）中负责恶意代码检测的神经元，并利用因果干预验证归因结果，从而为神经元级编辑、选择性遗忘等可靠性防御机制奠定基础。

## 2. 方法论
- **核心思想**：对 LLM 的 FFN 层神经元进行归因，识别哪些神经元促进（facilitator）或抑制（inhibitor）恶意代码的正确分类，然后通过因果干预改变这些神经元的活动值，观察模型性能变化。
- **关键技术细节**：
  - **归因方法**：
    - **集成梯度（IG）**：沿输入到输出的插值路径累积梯度，计算每个神经元对目标函数 \( F(x) = Z_{\text{correct}} - Z_{\text{incorrect}} \) 的贡献（\( Z \) 为原始 logit）。
    - **神经元电导（NC）**：扩展集成梯度，测量神经元从输入到输出传导的信号量。
  - **评分与筛选**：对每个样本，选取贡献最高（\( G_j \)）和最低（\( B_j \)）的 \( z \) 个神经元；剔除跨样本既出现在 \( G_j \) 又出现在 \( B_j \) 的模糊神经元；最后计算聚合电导得分（ACE），选出前 \( K=100 \) 个“好”神经元和“差”神经元。
  - **因果干预**：对好神经元和差神经元施加乘性缩放，设置五种条件：
    - 基线（无修改）
    - 抑制好（好神经元缩放=0，差神经元缩放=1）
    - 抑制差（好=1，差=0）
    - 增强（好=2，差=0）
    - 退化（好=0，差=2）
- **公式/算法流程**：论文中给出了 IG 和 NC 的数学定义，以及示例得分、ACE 的计算公式。整体流程包括数据集构建、代理提示、归因、评分筛选、因果干预评估。

## 3. 实验设计
- **数据集**：来自 PyPI Malregistry 的 1,500 个恶意包和 1,500 个良性包，共 3,000 个样本。按代码行数分桶（≤20, 21-50, 51-100, 101-200, >200），确保长度匹配。仅提取纯净恶意载荷（如 setup.py 或可疑的 __init__.py）。
- **基准（Benchmark）**：无公开标准基准，主要对比三种模型在六种干预条件下的准确率和 F1 分数，以及两种归因方法（IG vs. NC）的效果。
- **对比方法**：主要对比 IG 和 NC 两种归因方法的性能，以及不同干预条件之间的差异。
- **评价指标**：准确率（含 95% Wilson 置信区间）、每类 Precision/Recall/F1、McNemar 检验（配对显著性）、效果差距 Δ = Accuracy<sub>增强</sub> − Accuracy<sub>退化</sub>。

## 6. 主要结论与发现
- **归因方法有效性**：IG 和 NC 都能识别出具有因果作用的神经元，但效果高度依赖模型。
  - 对于低基线模型 Mistral-v0.3 (66.5%)，仅抑制差神经元（Suppress Bad）就能将准确率提升至 92.0%，增强干预更达到 94.1%。
  - 对于高基线模型 Llama3.1 和 Qwen2.5 (93.9%)，增强干预几乎无增益，但退化干预使 Llama3.1 准确率骤降至 50.0%（完全坍缩），Qwen2.5 则相对鲁棒（仅降至 83.2%）。
- **神经元分布**：每个模型将病毒检测信号集中在特定的层带：Llama3.1 和 Mistral 集中于中间层（层 5-14），Qwen2.5 集中于最后几层（层 25-26）。好神经元和坏神经元共定位在同一层带内。
- **稳定性**：神经元级的选择不可重复（Jaccard 中位数 <0.20），但层级别分布相对稳定（余弦相似度中位数 0.6-0.9）。Mistral 层级最稳定，Qwen2.5 最分散。
- **方法对比**：IG 在所有模型上产出相等或更大的效果差距 Δ，且对 Mistral 效果更显著；NC 效果普遍较弱。

## 7. 优点
- **创新性**：首次系统性地将机械可解释性方法（归因+因果干预）应用于 LLM 恶意代码检测的内部机制研究，为后续安全工作提供新视角。
- **方法论完整**：包含数据集构造、归因、筛选、因果验证及稳定性分析，逻辑链条清晰。
- **可复现性**：提供公开的匿名复制包（包含代码和数据），促进开放科学。
- **实际意义**：研究结果支持轻量级早期退出安全检测、层特异性模型编辑和神经元级选择性遗忘等实用防御手段。
