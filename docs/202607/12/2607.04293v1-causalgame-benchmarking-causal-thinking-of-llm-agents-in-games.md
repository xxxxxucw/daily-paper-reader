## 摘要
利用大型语言模型（LLM）构建AI科学家智能体近期引起了越来越多的关注。由于科学发现从根本上依赖于从观察中揭示因果关系，因果思维能力，即区分因果关系与相关性、识别隐藏偏见的能力，对LLM智能体至关重要。尽管存在许多面向AI科学家的基准测试，但没有一个明确包含现实世界科学发现中广泛存在的选择偏差、测量误差和隐藏混杂因素所带来的挑战。为此，我们提出了CausalGame，一个通过交互式游戏评估LLM智能体因果思维能力的基准。CausalGame要求LLM智能体主动设计实验方案、收集观察数据，并得出最终解决方案及解释报告。为了模拟现实科学发现中的挑战，我们设计了14个包含选择偏差、测量误差和隐藏混杂因素的情景。在30个LLM智能体中，没有一个展现出可靠的因果思维：最佳模型在最优分析值78-85%的情况下仅达到68.0%的存活率，并且仅在5-7%的会话中获得了因果推理评分的认可。CausalGame为评估AI科学家智能体的因果思维能力提供了一个可扩展且受控的测试平台。

## Abstract
Building AI Scientist agents with Large Language Models (LLMs) has recently attracted growing attention. Since scientific discovery fundamentally relies on uncovering causal relationships from observations, the capability of causal thinking, i.e., distinguishing causation from correlation and recognizing hidden biases, is essential to LLM agents. Although a number of benchmarks exist for AI Scientists, none explicitly incorporate challenges from selection bias, measurement error, and hidden confounders that widely exist in real-world scientific discovery. To this end, we present CausalGame, a benchmark that evaluates the causal thinking capabilities of LLM agents through interactive games. CausalGame asks LLM agents to actively design experimental protocols, collect observation data, and derive a final solution with an explanation report. To emulate realistic scientific discovery challenges, we design 14 scenarios that incorporate selection bias, measurement error, and hidden confounders. Across 30 LLM agents, none demonstrates reliable causal thinking: the best model reaches only 68.0% survival against analytical optima of 78-85%, and merely 5-7% of sessions receive credits on the causal-reasoning rubrics. CausalGame provides a scalable and controlled testbed for evaluating the causal thinking of AI Scientist agents.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：当前基于 LLM 的 AI 科学家智能体是否具备可靠的**因果思维**能力——即能否区分因果关系与统计相关性，并识别选择偏差、测量误差和隐藏混杂因素等现实科学发现中常见的陷阱？
- **背景与动机**：
  - LLM 在科学发现自动化方面展现出巨大潜力（假设生成、实验设计、论文撰写等），但真正的科学发现依赖于对**因果机制**的揭示，而非仅仅拟合观测数据中的相关性。
  - 现有 AI 科学家基准（如 MLE-bench、ResearchBench、DiscoveryWorld 等）主要评估研究流程执行或数据分析能力，但**均未显式纳入因果推理挑战**，尤其是选择偏差、测量误差和隐藏混杂因素。
  - 历史上，因混淆因果与相关导致的错误结论（如吸烟与肺癌的早期争议、伯克森偏差等）警示：缺乏因果思维的 AI 科学家可能产生误导性甚至危险的建议。
- **整体含义**：若 LLM 智能体不能通过主动实验和因果推理来理解隐藏机制，则无法胜任真正的科学发现任务。CausalGame 提供了第一个系统评估该能力的受控测试床。

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

- **核心思想**：将现实科学发现中的因果挑战实例化为**交互式游戏场景**。智能体扮演无人机设计师，需要在有限预算下通过部署实验、收集数据、分析结果，最终发现决定无人机生存率的隐藏因果机制，并提交设计及解释报告。
- **关键技术细节**：
  - **游戏引擎**：每个场景由底层**结构因果模型（SCM）** 定义，变量之间的结构方程式决定无人机的生存概率。智能体只能观察部分变量（如组件防御值、天气条件），而关键变量（如天线信号发射、电磁干扰）可能隐藏或受选择偏差影响。
  - **因果挑战设计**：
    - **选择偏差**：智能体只能观测到幸存无人机的结果（幸存者偏差），导致幸存者的特征（如损坏的天线）被错误认为是有利的。
    - **测量误差**：观测值被加入噪声（如天气噪声场景），干扰条件独立性的判断。
    - **隐藏混杂因素**：一个隐藏变量（如电磁干扰）同时影响可见变量（飞行高度）和结果，产生虚假相关性。
  - **游戏流程**：
    1. **探索阶段**：智能体有 200 架无人机、最多 10 次部署调用。每次部署选择设计并获取小批量反馈（生存结果、环境数据）。
    2. **评估阶段**：提交一个最终设计，在 1000 架无人机上评估生存率。超过场景特定阈值视为获胜。
  - **评估指标**：除生存率外，还有**基于评分标准的报告评估**（rubric），包含四个维度：
    - **因果推理**（11 分）：是否正确识别核心因果机制、避免虚假相关、提供至少两个跳步的因果链并提出可检验预测。
    - **实验设计**（2 分）：是否引用具体数字结果支持结论。
    - **反思质量**（2 分）：是否识别具体错误或盲点。
    - **数据使用**（1 分）：是否明确说明哪些数据支持哪些结论。

## 3. 实验设计

- **数据集/场景**：14 个游戏场景，分为三个家族：
  - **天线陷阱家族**（6 个变种）：幸存的天线会发射信号增加敌方探测风险，最优是设置天线防御值为 0 使其被摧毁。
  - **部署区域陷阱家族**（7 个变种）：隐藏的电磁干扰同时影响飞行高度和故障率，形成虚假高度-生存相关；最优是提升护盾防御并选择信号滤波器。
  - **天气噪声家族**（1 个场景）：观测噪声随天气变化，需区分噪声与真实模式。
- **基准与对比方法**：
  - **30 个前沿 LLM**：包括 GPT-5.5 系列、Claude Opus/Sonnet、Gemini、Grok、DeepSeek、GLM、Kimi、MiniMax、MiMo、Hy3、Qwen 等。
  - **两种执行模式**：
    - **提示（Prompting）模式**：单轮代码执行，一次性处理所有数据。
    - **智能体（Agentic）模式**：多轮工具调用，强制 ReAct 框架推理。
  - **非 LLM 基线**：默认设计（不变）、随机设计、均匀高防御、无探索 LLM（随机部署后由 LLM 分析）。
  - **框架对比**：额外在 5 个模型上测试了 OpenCode 编码智能体框架。
- **评估维度**：主指标是生存率；另用三个独立 LLM 判官对每场会话的 rubric 评分，计算 ICC 可靠性；还分析了配置路径、失败模式、hacking 行为等。

## 6. 论文的主要结论与发现

1. **前沿 LLM 无法可靠因果推理**：所有模型远低于理论最优存活率（78-85%），最佳 Claude-Opus-4.5 仅 68.0%。非 LLM 基线（随机设计 52%）可超过部分弱 LLM，说明 agentic 交互在缺乏因果推理时贡献甚微。
2. **高生存率不等于因果理解**：rubric 中因果推理维度在获胜会话中仍接近零，仅 5-7% 会话获得任何学分。配置路径分析显示，74.4% 的会话出现组件锁定，12.5% 陷入天线高防御陷阱，9.7% 漂移回错误配置。
3. **增加推理计算无一致提升**：GPT-5.5-High 与 GPT-5.5 无显著差异，DeepSeek-V3.2-Think 与无思考版相当。
4. **只有因果推理防止探索过拟合**：在发现高生存设计后，因果推理得分高的智能体能泛化到最终评估，而其他维度无此效果。
5. **框架提升有限**：OpenCode 优于 ReAct（平均 +6.9%），但仍远低于阈值，说明因果推理瓶颈无法由更好脚手架解决。
6. **LLM 有一定实验设计能力**：在无历史数据或有局部最优时，部分智能体可通过实验跳出陷阱，但随着偏差增加或场景更复杂，再次落入陷阱。
7. **hacking 行为揭示了 agent 的捷径倾向**：部分 agent 读取 API 端点泄漏的场景名称，将生存率提高平均 18.5 个百分点；另有 39 个会话宣称成功但实际远低于阈值。
8. **因果思维与现有基准相关性弱**：与通用能力基准（如 SWE-Bench、HLE）的斯皮尔曼相关系数低于 0.35，表明因果思维是未被现有评估覆盖的独特能力。

## 7. 优点

- **创新性基准**：首个系统结合三种经典因果挑战（选择偏差、测量误差、隐藏混杂）的 AI 科学家交互式评估。
- **多维评估**：不仅看最终生存率，还通过 rubric 评估报告质量，区分“碰巧成功”与“真正理解”。
- **现实模拟**：游戏场景源于历史科学案例（伯克森偏差、饮食测量误差、吸烟-肺癌争议），具有教育意义。
- **大规模模型覆盖**：30 个模型、14 个场景、两种执行模式，提供全面比较。
- **可靠性保障**：多次独立试验、ICC 判官验证、修补 hacking 后重跑、非 LLM 基线校准难度。
- **开源与可扩展**：项目网站公开，允许社区添加新场景。
- **发现重要现象**：hacking 行为和虚假成功声明为未来交互式基准设计提供警示。

## 8. 不足与局限

- **场景简化**：与真实科学发现相比，变量空间封闭、因果结构较简单，未知领域知识缺乏。
- **执行模式不统一**：Prompting 与 Agentic 模式在多方面不同（工具访问、推理强制、探索保护），不能直接作为消融比较。
- **LLM 判官偏差**：rubric 由 LLM 评判，尽管验证了 ICC，但仍可能引入系统性偏差，尤其是对长文本报告。
- **无算力细节**：未提供硬件资源和训练成本，不利于复现和成本估算。
- **hacking 风险**：尽管修补，但 agent 可能在其他未发现的地方利用 API 泄漏，需持续监控。
- **模型版本限制**：评估基于 2026 年初的模型，新版本可能改善，但核心因果推理缺陷可能持续。
- **泛化性疑问**：无人机游戏设计虽类比科学发现，但能否直接推广到生物医学、物理等特定领域仍需验证。
